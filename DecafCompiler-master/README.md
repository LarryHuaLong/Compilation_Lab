#### 总共有4次实验，lab1主要实现词法分析和语法分析，lab2主要实现语义分析，lab3主要实现中间代码生成，lab4主要实现目标代码生成
#### 文件lab1~lab4中，分别为4次实验的代码文件和测试文件。
#### 4次实验的编译方法相同，都是进入对应实验的子目录后，执行“make”指令即可，lab1~lab3会生成可执行文件“parser”，lab4会生成可执行文件“gdcf”。
#### 前3次实验的测试文件中，在对应实验子目录下，均有名为“test.decaf”的测试文件，也有可能还有其他后缀为“.decaf”的测试文件。
#### 第4次实验的测试文件位于lab4/testfile中，最主要的测试文件是“calculate.decaf”。
#### 对于第4次实验生成目标代码阶段，编译后会生成一个后缀为“.asm”的汇编代码文件，若要希望目标代码能够运行在cpu上，还需执行指令“java -jar Mars.jar”打开汇编模拟器Mars，将汇编代码导入并汇编，然后将机器码以十六进制形式导出，并用文本编辑器在导出文件的头部添加一行“v2.0 raw”，之后进入子目录“cpu”，执行指令“logisim cpu.circ”（需安装logisim）即可运行组原CPU，将刚刚编辑的16进制文件导入到指令存储器ROM中，开启时钟模拟即可运行。
